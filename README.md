# AI-Ready Sidewalk Accessibility Dataset for Koreatown, Los Angeles

This project turns public pedestrian-infrastructure data for Koreatown, Los Angeles into a QGIS-ready review dataset for geospatial AI workflows. I cleaned and validated sidewalk, curb-ramp, driveway, crosswalk, intersection, street-centerline, and transit-stop layers, then added transparent QA flags and heuristic labels to help reviewers spot likely gaps, conflicts, and features that deserve closer inspection.

This is not an ADA compliance audit. The labels are practical map-data review signals based on public GIS geometry, attributes, and proximity rules.

Live demo: https://btafreshian.github.io/koreatown-sidewalk-accessibility/

Interactive map: https://btafreshian.github.io/koreatown-sidewalk-accessibility/interactive_accessibility_map.html

## What This Contains

- A reproducible Python pipeline for downloading public GIS and GTFS data, building the Koreatown AOI, cleaning geometries, assigning labels, and exporting QGIS-ready assets.
- A public GitHub Pages portfolio view in `docs/`, including a static landing page and an interactive map.
- Local QGIS outputs, static maps, QA tables, and source-count summaries generated by the pipeline.
- Documentation for data sources, methodology, labeling logic, QGIS usage, and project limitations.

## Data Sources

- LA City NavigateLA ArcGIS REST MapServer: sidewalks, access ramps, driveways, curbs, crosswalks, intersections, parkways, alley sidewalks, sidewalk boundaries, and street centerlines.
- LA Times Neighborhood Boundaries FeatureServer: Koreatown AOI polygon.
- LA Metro GTFS: bus and rail stops.
- OpenSidewalks schema: reference inspiration only, not a strict validation target.

See [docs/data_sources.md](docs/data_sources.md) for endpoints and fallback behavior.

## Reproduce Locally

.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python -m src.pipeline

```

make install
make all
make test
make clean
```

## Labels

The main sidewalk layer assigns one of five review labels:

- `accessible`
- `missing_ramp`
- `disconnected`
- `obstacle_or_driveway_conflict`
- `needs_review`

The labeled layer also keeps the underlying issue flags and `label_reason` values, so each label can be traced back to the rule that produced it.

Label counts from that run:

- 6,982 `accessible`

These counts are a snapshot from the most recent local pipeline run. They may change as public source data changes.

## Limitations

- Labels are heuristic review signals, not legal ADA findings.
- The connectivity graph is an approximation built from nearby/touching sidewalk polygons, not a routable pedestrian network.
- Public source data can be incomplete, stale, or spatially generalized.
- Ramp presence comes from available source attributes and proximity rules, not field verification.
